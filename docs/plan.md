# Project Plan: Beginner's Guide to American Mahjongg Website

## Project Overview
This project involves creating a simple two-page website that introduces beginners to American Mahjongg. The website will include a homepage, an about page, and one shared CSS file for consistent styling.

## Work Breakdown Schedule
1. Update README.
2. Create docs folder.
3. Upload project scope, making sure to incorporate changes from received feedback.
4. Create project plan.
5. Build homepage. 
6. Create about.html with setup, tile types, and basic gameplay.
7. Build about page.
8. Add styling. Create style.css and apply consistent styling to both pages.
9. Test website by checking layout, links, and live GitHub site.
10. Complete reflection. Create retrospective.md answering what went well, what went wrong, and what I would do differently.
11. Submit project with correct GitHub URL in Canvas.

## Risk Analysis Using TAME Framework

### Time

One risk is not having enough time to complete all required files due to full-time employment, other coursework, and other personal responsibilities. 

**Mitigation**: I will use a checklist, complete one file at a time, and test the website before submission.

### Ability

One risk is my beginner-level knowledge of HTML, CSS, and GitHuB.

**Mitigation**: I will keep the website simple, using concepts covered in class, and avoid advanced features that are outside the project scope.

### Budget

There is no financial risk because this project uses free resources.

**Mitigation**: I will only use free resources, including course materials, VS Code, and GitHub.

### Expectations

One risk is misunderstanding the final project requirements.

**Mitigation**: I will compare the finished project against the final project checklist and submission requirements to make sure the README, docs folder, website files, and live site are complete.
