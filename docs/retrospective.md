# Project Retrospective: Beginner's Guide to American Mahjongg Website

## What Went Well

I am proud that I was able to build a complete project that included both planning documentation and a working two-page website. I was able to create a clear topic, organize the website around beginner-friendly content, and connect the homepage and about page with working links.

This project helped me understand how planning documents influence a website. The scope statement outlined what the website should include, and the project plan helped me stay focused on the required files and requirements for final submission.

Making small improvements along the way also helped. I added styling, adjusted colors, and improved spacing. Then, I decided to include an image to illustrate the main tile types. These changes made the website more engaging for viewers.

## What Went Wrong

At first, I was not entirely sure whether I was supposed to overwrite my existing GitHub repository or create a new repository. Once I understood what it meant for the final project to be housed within a single public repository, I updated my existing repository so the live site would display the final Mahjongg project.

I also had to figure out how to create and use the docs folder correctly. At first, it was not obvious how GitHub handles folders, but I finally figured out that a folder has to contain a file before it appears in the repository. This took some trial and error.

Another challenge was using GitHub confidently. I am still not completely confident in GitHub or in trusting that commits are working the way I expect. I had to check the live site and repository several times to make sure the files were actually updating.

I also wish I had more time to play with styling and images. The website works and meets the assignment requirements, but with more time I would have liked to improve the visual design.

## What I Would Do Differently

If I were to do this project again, I would start by setting up the repository structure, creating the docs folder, website files, image folder, and README before adding most of the content. This would have made me feel more organized from the beginning.

I would also give myself more time to work with CSS. I was able to apply consistent styling, but I would like to have played more with layout, spacing, images, and color choices so the site feels more professional.

Finally, I would have liked to spend more time working with GitHub. I understand more now than I did at the beginning, but I would like to be more comfortable with commits, file locations, and how changes appear on the live GitHub site.
